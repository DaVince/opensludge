Anti-aliasing

When a character is drawn to the screen, it uses the character's current anti-aliasing settings, clamped so they don't exceed the game's current maximum anti-aliasing settings. By default, a character's anti-aliasing settings (and the maximum anti-aliasing settings) will be the ones defined using the character rendering settings dialog box in the project manager. You can also change the settings using setCharacterAntiAliasing and setMaximumAntiAliasing.

With anti-aliasing enabled (especially with the horizontal and vertical softness values set to higher values) drawing a character to the screen can take a little longer than drawing the character to the screen with anti-aliasing disabled. This is even more apparent with large sprites or with sprites scaled up to cover large parts of the screen. It's therefore recommended that if you're going to use anti-aliasing in your game, you give the player the opportunity to enable and disable it, and perhaps even choose the graphics quality themselves (eg. a key which cycles through setting the anti-aliasing to off, on with softness 0 and on with softness 4, or a slider in the game's pause menu). Because it's also possible for the game to get the (on or off part of the) initial anti-aliasing setting from the game's .ini file, this functionality could also be added to a seperate startup menu which writes out the appropriate .ini file and then launches your game.
