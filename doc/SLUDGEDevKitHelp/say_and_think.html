say, think and ponder

Syntax:

say (thisObject, text);

say (thisObject, text, soundFile);

think (thisObject, text);

think (thisObject, text, soundFile);

ponder (thisObject, text);

Purpose:

Say makes thisObject say the contents of the text parameter. The text is shown on the screen in the speech colour defined in the object type definition. If a soundFile parameter is specified, the sound may be used instead of (or as well as) the text - this depends on the current setSpeechMode setting.

If thisObject is a character:

Any displayed text will be shown over the head of the character and the character's current animation will be changed to the speaking animation (for the character facing its current direction) defined in the costume assigned to the character. When the character has finished speaking, the character's animation will change to the standing still animation.

If thisObject is a screen region:

Any displayed text will be shown above the last region to have been added which represents the object; that is, if there are three screen regions which represent the same object and the object is told to speak, the text will appear over the third region.

If thisObject isn't a character or screen region:

Any displayed text will appear centred at the top of the screen.

The function from which the say command is called will pause until the object stops speaking. The time this takes depends on the number of characters in text and is calculated by the SLUDGE engine at runtime. If the soundFile parameter is specified (and used), the time for which the function pauses is the length of time needed to play the sound.

Valid file formats for the soundFile parameter are Windows wave files (.WAV) and Ogg Vorbis (.OGG), as in the playSound function.

The difference between say, think and ponder

The above text describes say, which is the standard function for making characters talk. think and ponder exist for when you want more control.

think doesn't automatically animate the character, which allows you to put in a custom animation for the speech.

ponder doesn't animate the character, and does not pause the calling function. The text will remain until the same character is made to speak, think or ponder again (ponder with an empty string to stop pondering), or the character is removed, or normalCharacter is called. With ponder, it is possible to have several people pondering at the same time. Also note that because ponder does not stop the action, it's not affected by skipSpeech. You'll have to take care of that yourself too. Does this seem complicated? Don't worry - in most cases, you'll never need to use ponder.

These three examples do almost the same thing:

say (woman, "Hello!");

animate (woman, jumpingOfJoyAnimation);
think (woman, "Hello!");
normalCharacter (woman);

ponder (woman, "Hello!");
pause (animate (woman, elaborateJumpingOfJoyAnimation));
normalCharacter (woman);

Return value:

No return value.
