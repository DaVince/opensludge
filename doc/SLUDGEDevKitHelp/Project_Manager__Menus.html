Project Manager: Menus

The SLUDGE project manager features four menus: File, Project, SLUDGE Settings and Help. Each menu contains a number of options, described below.

File menu:

New SLUDGE project:

Creates and loads a new, empty project file. The same as hitting the Create new project button in the Project management section of the window.

Open SLUDGE project:

Loads an existing project file. The same as hitting the Open existing project button in the Project management section of the window.

Save:

Saves the current project, overwriting the file from which it was loaded. The same as hitting the Save project button in the Project management section of the window.

Save as:

Lets you save the existing project to a different filename. Note that if you save the project file to a directory other than the directory in which the old file was located, the file locations listed in the Files in this project will be wrong. The program will warn you of this at the time.

Revert:

Reloads the current project, undoing all of your changes since the last time it was loaded or saved. This will NOT undo changes to the script files listed in the Files in this project section, only the contents of the list and any changes made in the Project settings section and the Project properties.dialog box.

Close:

Closes the current project. The program will return to its "No project loaded" state.

Exit:

Quits the SLUDGE project manager.

Project menu:

Add file:

Adds a SLUDGE script (or definition) file to the project. The same as hitting the Add file button in the Files in this project section of the window.

Find in files:

Lets you search the files in your project for text - useful for finding where you defined a particular function or object. The same as hitting the Find in files button in the Files in this project section of the window.

Project properties:

Opens the Project properties dialog box. The same as hitting the Project properties button in the Project management section of the window.

Compile:

Compiles the current project. The same as hitting the Compile project button in the Project management section of the window.

Compile and run:

Compiles the current project and, if the compilation is a success, runs the compiled game. The same as hitting the Compile and run project button in the Project management section of the window.

SLUDGE Settings menu:

Keep compressed images:

If this option is checked, the compressed images generated during compilation will be kept on your machine after compilation. This will take up extra space but will decrease the time it takes to compile a project, as during subsequent compilations any images which have not changed do not need to be recompressed.

Write strings to text file:

During compilation, generate a file containing all the strings used in your game. Useful if you want to spellcheck the strings without also spellchecking your SLUDGE code.

Verbose errors (development mode):

Include function names and other debugging information in your compiled game. This means that run-time error messages are more meaningful. It's recommended that you keep this option checked while developing a game, and uncheck it before compiling a release version.

File locations:

Lets you choose which programs you'd like the SLUDGE project manager to call when handling the various file types with which it has to deal. Currently, the only file you can set is your choice of text editor.

Window sizes:

Lets you choose the width and height of the windows used by the SLUDGE z-buffer maker and SLUDGE floor maker programs.

Help menu:

About SLUDGE:

Displays a pop-up dialog box giving the version number of the project manager and a link to the Hungry Software SLUDGE website. The same as hitting the About button in the Project management section of the window.

SLUDGE documentation:

Opens this help file. The same as hitting the Help! button in the Project management section of the window.
