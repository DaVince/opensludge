launch and launchWith

Syntax:

launch (filename);

launchWith (program, filename);

Purpose:

Launches a file which isn't contained within the compiled SLUDGE game, such as a help document or an order form. The filename parameter must be a string, not a file handle. The filename given should be relative to the location of the compiled SLUDGE game; that is, if the SLUDGE game is "C:\Games\SLUDGE\Here\My game.SLG", calling launch with a parameter of "docs\hello.html" would try to run "C:\Games\SLUDGE\Here\docs\hello.html". This is so that, if you distribute your game, it can be installed anywhere on a machine and the files - so long as they're distributed with the compiled game - can still be found and launched.

If you're using the launchWith function, you can also specify what program you want to use to open up a file. For example, you may have a file called "Buy me online.html" which you want to launch in notepad (why you'd want to do this, I'm not sure) rather than the default application on someone's system (which, for an HTML file would probably be Internet Explorer, Opera or Netscape). Simply use the launchWith function, giving the program parameter as "notepad.exe" and the filename parameter as "Buy me online.html".

A more realistic use for launchWith is starting a SLUDGE game from within a SLUDGE front-end menu system when you've got a renamed engine in the same folder. Check out the SLUDGE examples for more details of how to do this.

These functions support filename bodging - for details see the bodgeFilenames command.

Return value:

The function returns TRUE if the file could be launched and FALSE if not.
